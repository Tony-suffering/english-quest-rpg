## 本家 Anki の仕組み (調査結果)

本家 Anki は SM-2 アルゴリズムの改良版を使用。Leitner Box とは根本的に違う。

### カードの4状態

| 状態 | 説明 |
|------|------|
| **New** | 未学習。ease factor なし、interval なし |
| **Learning** | 初回学習中。固定ステップ (デフォルト 1分→10分) を進む |
| **Review** | 卒業済み。指数的にインターバルが伸びる (SM-2) |
| **Relearning** | Review カードを Again した時。再学習ステップを踏む |

### デフォルトパラメータ

| パラメータ | デフォルト値 |
|-----------|-------------|
| 学習ステップ | 1分, 10分 |
| 卒業インターバル | 1日 |
| Easy インターバル | 4日 |
| 初期 ease | 250% |
| ease 下限 | 130% |
| Hard 倍率 | 1.2x |
| Easy ボーナス | 1.3x |
| ラプス新インターバル | 0% (リセット) |
| リーチ閾値 | 8回ラプス |
| 新規カード/日 | 20 |
| 最大レビュー/日 | 200 |

### ボタンの挙動 (Review カード)

| ボタン | ease 変化 | インターバル計算 |
|--------|----------|-----------------|
| Again | -20pt | Relearning へ。interval = old * 0% = 1日リセット |
| Hard | -15pt | (interval + 遅延/4) * 1.2 |
| Good | 変化なし | (interval + 遅延/2) * (ease/100) |
| Easy | +15pt | (interval + 遅延) * (ease/100) * 1.3 |

### Learning カードのボタン

| ボタン | 挙動 |
|--------|------|
| Again | 最初のステップに戻る |
| Hard | 現在のステップを繰り返す |
| Good | 次のステップへ。最終ステップなら卒業 (1日) |
| Easy | 即座に卒業 (4日) |

### Leitner Box との根本的な違い

| | Anki (SM-2改) | Leitner Box (現在の実装) |
|-|---------------|------------------------|
| インターバル | カードごとに連続値 (1,3,8,21,54...) | Box固定 (1,3,7,14,30日) |
| 難易度追跡 | カードごとの ease factor (130-300%+) | なし。同じBoxは同じ間隔 |
| 失敗時 | ease -20pt、interval * new_interval | Box 1 に完全リセット |
| 応答粒度 | 4段階、各々でインターバル・ease変化が違う | 実質2値 (正解/不正解) |
| 成長 | 指数的・適応的 | 固定・線形 |
| 遅延ボーナス | あり (遅れて正解→長いインターバル) | なし |

## 現在の実装の何がダメか

### 根本問題: Leitner Box を使ってる

既存 `srs-engine.ts` は Leitner Box (1-5) + SM-2 ease factor の混合。
本家 Anki とは以下が違う:

1. **Learning/Relearning 状態がない** -- New → いきなり Box 1 (1日後)。本家は 1分→10分の学習ステップがある
2. **インターバルが Box 固定** -- Box 3 は必ず7日。本家はカードごとに異なる
3. **失敗で Box 1 リセット** -- 100日間隔のカードも1日に戻る。本家は new_interval 設定で調整可能
4. **遅延ボーナスなし** -- 3日遅れて正解しても評価されない
5. **Hard が「1つ下のBox」** -- 本家は interval * 1.2 (微減)

### UI 問題

1. **カレンダー形式ではない** -- 他のページ (phrases, goroku, word-review) は全部カレンダー
2. **初回10枚制限** -- 1,044枚中10枚/日。体験が壊れてる
3. **表裏が逆かも** -- 英語→日本語より、日本語→英語の方が recall 訓練になる
4. **設定変更できない**
5. **モバイル対応が甘い**

## カレンダー形式への設計方針

### 既存カレンダーの共通パターン (goroku, phrases, word-review)

- 7列グリッド (日-土)、`gap: 3px`
- 月ナビゲーション (矢印 + 年月表示 + 今日ボタン)
- セルの色分け: 未着手 / 進行中 / 完了
- 今日: 金色ボーダー (`#D4AF37`)
- 右サイドパネル (320-540px): 選択日の詳細
- モバイル: 縦スタック

### Anki カレンダーでの適用案

```
┌─────────────────────────────────────────────────────┐
│  ◀  2026年2月  ▶  [今日]                            │
├────┬────┬────┬────┬────┬────┬────┤
│ 日 │ 月 │ 火 │ 水 │ 木 │ 金 │ 土 │
├────┼────┼────┼────┼────┼────┼────┤
│    │    │    │    │    │    │ 1  │
│    │    │    │    │    │    │○12 │
├────┼────┼────┼────┼────┼────┼────┤
│ 2  │ 3  │ 4  │ ...                               │
│●38 │○25 │    │                                    │
└────┴────┴────┴─────────────────────┘
     ● = セッション完了  ○ = due あり未完  空 = due なし
     セル内の数字 = その日の due/reviewed 数
```

右パネル (選択日):
- その日の due カード一覧
- カードごとの box/ease/interval 情報
- 「この日のレビューを開始」ボタン
- セッション履歴 (もし完了済みなら結果表示)

### カレンダーセルに表示する情報

| 状態 | セル表示 |
|------|---------|
| due カードあり + 未レビュー | 薄い金色背景 + due数 |
| セッション完了 | 緑背景 + reviewed数 |
| 今日 | 金色ボーダー |
| due なし | 白 |
| 未来 | グレー (予定 due 数プレビュー) |
