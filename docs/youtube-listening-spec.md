# YouTube Listening システム仕様書

## 概要

日本語のYouTube動画を素材に、言語ギャップが面白い表現を10個厳選して学ぶシステム。
カレンダーの1日 = 1動画 = 10表現。

---

## データ構造

### ListeningContent (src/data/english-listening.ts)

```typescript
interface ListeningContent {
    id: string;           // slug: "rensox-whitesox-2026-rebuild"
    youtubeId: string;    // YouTube video ID (空でもOK)
    title: string;        // 日本語タイトル
    description: string;  // 1行の説明
    date: string;         // カレンダー日付 "2026-02-25"
    segments: ScriptSegment[];   // 英語動画: 全文スクリプト / 日本語動画: 空配列
    expressions: ExpressionPick[]; // 厳選10個
}

interface ExpressionPick {
    japanese: string;   // 元の日本語表現
    english: string;    // ネイティブが同じ場面で言う英語
    context?: string;   // 動画内での使用文脈（原文引用）
    why?: string;       // なぜこの表現が面白いか（言語ギャップの解説）
}
```

### 2種類のコンテンツ

| タイプ | segments | expressions | 例 |
|--------|----------|-------------|-----|
| 英語動画 | 全文バイリンガル | 10個 | Kyle Tucker、Tyler Glasnow |
| 日本語動画 | 空配列 | 10個 | RenSoxシリーズ |

## 表現選定プロセス（Claude向け）

### Step 1: 全文を読んで候補を25-30個拾う

- 内容（選手の成績、数字）ではなく **言い回し** だけを見る
- 野球用語そのものではなく、日常でも使える表現を優先
- 候補はコメントブロックに全部記録する

### Step 2: 除外フィルター

以下に該当するものを除外:

| 除外理由 | 例 |
|----------|-----|
| カタカナ英語の逆輸入 | プロスペクト、ハイレバレッジ、FIP |
| 直訳で済む | 潜在能力=potential、飛躍=leap |
| 文化ギャップ（言語ギャップではない） | 温水洗浄便座、お尻が痒い |
| 単語レベルで薄い | 完全に無理、卓越した |
| 内容依存で汎用性なし | 白い靴下になってしまった |
| 前回のエントリーと被る | 同カテゴリの表現が既出 |

### Step 3: 最終10個の選定基準

1. **日本語の感覚入口が英語にない** -- 匂い→直感、兆し→占い、ど→強調
2. **身体・五感メタファーの差** -- 含む=口、匂い=鼻、均す=土木
3. **汎用性** -- 野球以外の文脈でも使える表現
4. **ジャンルの分散** -- 10個の中で同カテゴリが被らない
5. **whyフィールドの面白さ** -- 「へえ」と言わせられるか

### Step 4: コードに選定プロセスをコメントで残す

レッズ2026エントリーのフォーマットに従う:

```typescript
// ===== [チーム名]2026 選定プロセス =====
//
// ■ 候補として拾った表現 (N個):
// ■ 除外した理由:
// ■ 選んだ10個の基準:
// ■ 迷って落としたもの:
// =====
```

## whyフィールドの書き方

### 良い例（レッズ Day 20 baseline以降）

> 土壇場の元ネタは処刑台。首が飛ぶ直前。英語のeleventh hourは時計の11時。命がかかってるvs遅刻しそう。緊迫感のスケールが3段階違う

- 日本語の語源 → 英語の語源 → 差の正体
- 短文。句読点で切る。体言止め可。
- 最後の一文がオチ（比較の結論）

### ダメな例

> この表現は日本語特有のニュアンスがあり、英語では完全に翻訳することが難しいです。

- 教科書トーン禁止
- 「翻訳が難しい」で終わるな。なぜ難しいかを具体的に書け
- 居酒屋の友達に説明する口調

## ページ構成

### カレンダーページ (/english/listening)

- 月カレンダー表示、日付にドットで動画有無を表示
- 日付クリック → 右パネルに動画タイトル + 表現リスト
- 表現をphrasesテーブルに自動登録（重複チェック付き）
- リストビュー切替あり

### プレイヤーページ (/english/listening/[id])

- 4モード: video（YouTube同期）/ tts / lesson / dual
- セグメント単位で英語+日本語表示
- YouTube IFrame API連携
- 保存フレーズ機能

---

## ワークフロー（新しい動画を追加する手順）

1. ユーザーがスクリプト（文字起こし）を貼る
2. Claudeが全文から候補を25-30個抽出
3. 除外フィルターを適用
4. 最終10個を選定（選定プロセスをコメントに記録）
5. `english-listening.ts` にエントリー追加
6. 日付はカレンダーの空き日に割当（1日1動画）

### 日本語動画の場合
- `segments: []`（空配列）
- `expressions` のみ記入

### 英語動画の場合
- `segments` に全文バイリンガルスクリプト
- `expressions` に厳選10個
