# 居酒屋TOEIC -- Design Document

> "TOEICがつまらないのは試験英語だから。ストーリーないし、だから何って内容しかない。"
> -- これを解決する。

---

## なぜ居酒屋TOEICが最強になるか

### TOEIC の構造的弱点（リサーチから判明）

1. **パラフレーズ認識が600→800の壁** -- 正解は音声と同じ単語を使わない。言い換えを見抜く力が全て
2. **間接回答がPart 2の罠** -- "When does the train leave?" → "Let me check the schedule." が正解になる
3. **ビジネス英語のくせに個性ゼロ** -- 誰が喋ってるかわからない。感情移入不可能
4. **日本人平均574点** -- 500-600帯で停滞する人が最も多い。ここを突破させる教材が存在しない
5. **リスニングの方がスコア効率がいい** -- 76/100正解でL395 vs R335。リスニング集中が正解

### 居酒屋が解決する理由

| TOEICの問題 | 居酒屋TOEICの解決策 |
|---|---|
| 誰が喋ってるかわからない | 常連キャラが毎回登場。声・性格・口癖がある |
| ストーリーがない | 居酒屋を舞台にした連続ドラマ。毎回前回の続きがある |
| ビジネス英語が退屈 | 会社帰りの居酒屋トーク = ビジネス英語が自然に出る |
| パラフレーズが見抜けない | キャラの口癖で言い換えパターンを体に染み込ませる |
| 試験対策が苦行 | 笑えるストーリーで「続きが気になる」教材にする |

## コンセプト: 「のれんの向こう側」

居酒屋「のれん」-- 東京の路地裏にある小さな居酒屋。
マスターは元TOEIC満点講師だが、教えるのが嫌になって居酒屋を開いた。
でも常連客がみんな英語の悩みを持ち込んでくるから、結局カウンター越しに教えてる。

### キャラクター（常連客）

| 名前 | 年齢 | 職業 | 英語レベル | 個性 | TOEIC的役割 |
|---|---|---|---|---|---|
| **マスター（権藤）** | 58 | 居酒屋店主（元TOEIC講師） | 990満点 | 口数少ない。でも核心を突く一言が鋭い | 解説役。でも説教臭くない |
| **ユキ** | 28 | 商社の営業 | 620→目標800 | 真面目だけどツッコミ鋭い。酒入ると毒舌 | メイン主人公。学習者の代弁者 |
| **タケシ** | 35 | IT企業のPM | 550→目標700 | お調子者。間違えるけど前向き | よくある間違いの体現者 |
| **リサ** | 32 | 外資系マーケティング | 860 | 帰国子女。英語は得意だけどTOEICの罠にハマる | 「英語できる≠TOEIC高い」の体現 |
| **ケンジ** | 45 | 建設会社の部長 | 480→目標600 | 昭和のおっさん。でも部下のために必死 | 初心者目線。感情移入ポイント |
| **ミナ** | 24 | 派遣社員 | 730 | 天然。でもリスニングだけ異常に強い | リスニング特化の秘訣を体現 |

### ストーリー構造

1話 = 1つのTOEICスキルを習得

物語の中で自然にTOEICの問題パターンが出てくる。
キャラ同士の会話がそのままリスニング練習になる。

**例: Episode 1 「ユキ、昇進がかかってる」**

```
[居酒屋のれん。金曜の夜。]

ユキ: マスター、ビール。あと愚痴聞いて。
マスター: （グラスを拭きながら）仕事か。
ユキ: 来月のTOEICで800取らないと、海外営業部に異動できない。
      今620なんだけど...180点上げるとか無理じゃない？
マスター: 無理じゃない。
ユキ: え？
マスター: お前、リスニングで何点だ。
ユキ: 320...
マスター: じゃあそこだ。リスニングは76問正解で395点になる。
         リーディングで同じ76問正解だと335点にしかならない。
ユキ: ...リスニングの方がコスパいいってこと？
マスター: そういうこと。

[タケシが入ってくる]

タケシ: おつかれー！何の話？
ユキ: TOEICの話。
タケシ: あー俺も受けるわ。でもさ、Part 2って何であんなに意味わかんないの？
       "When does the train leave?" って聞かれたら "At 3:30" でしょ普通。
       なんで "Let me check the schedule" が正解になんの？

マスター: （枝豆を出しながら）
         ...それがわかったら、お前はもう650超えてる。
```

→ この後、マスターがPart 2の「間接回答パターン」を、
  居酒屋のやり取りに例えて教える。

```
マスター: タケシ。ビール何にする？
タケシ: えーと、何があるんすか？
マスター: ...今お前、俺の質問に答えたか？
タケシ: え？ 答えて...ない？
マスター: 「何にする」に対して「何があるか」で返した。
         これが間接回答だ。自然だろ？
タケシ: ......あ。
ユキ: あー！だから "Let me check the schedule" が正解になるのか！
```

## TOEIC Part別 居酒屋コンテンツ設計

### Part 1: 写真描写 (6問)

**居酒屋版**: 「のれん」の店内写真を見て、正しい描写を選ぶ
- マスターがカウンターを拭いている写真
- 常連たちが乾杯している写真
- 厨房で料理を準備している写真

**学習ポイント**: 動作・位置・状態の英語表現
**トラップ訓練**: similar-sounding words、wrong preposition

### Part 2: 応答問題 (25問)

**居酒屋版**: 常連同士の会話を使った間接回答トレーニング
- 居酒屋の注文シーン → 間接回答の自然さを体感
- 仕事の愚痴 → WH疑問文 + 間接回答
- 酔った状態での受け答え → tag questions、statements

**学習ポイント**: 間接回答パターン認識
**キラーコンテンツ**: 「酔いレベル別回答パターン」
  - シラフ: 直接回答
  - ほろ酔い: やや間接
  - 酔っぱらい: 完全に間接（でもこれがTOEICの正解パターン）

### Part 3: 会話問題 (39問 = 13会話x3問)

**居酒屋版**: キャラクター同士のストーリー会話
- 2人会話: ユキとマスター、タケシとケンジ
- 3人会話: 常連3人の議論
- グラフィック付き: 居酒屋のメニュー表、予約表、シフト表

**学習ポイント**: パラフレーズ認識、話者の意図把握
**仕掛け**: 同じ話題を違う言い方で繰り返すキャラ設定
  - ユキは正確な表現を使う
  - タケシは砕けた言い方をする
  - → 同じ意味の異なる表現を自然に比較できる

### Part 4: 説明文問題 (30問 = 10話x3問)

**居酒屋版**: キャラの職場で流れるアナウンス・留守電を再現
- ユキの商社: 海外出張の変更通知
- タケシのIT企業: システムメンテナンスのお知らせ
- ケンジの建設会社: 安全研修の案内
- マスターの仕入れ先: 食材の配送変更

**学習ポイント**: 冒頭文で話者/聴衆/目的を掴む
**仕掛け**: 「これ、さっきユキが愚痴ってたやつの元ネタだ！」と繋がる

### Part 5-7: リーディング

**居酒屋版**: キャラの仕事で使う実際のメール・書類
- ユキの英文メール（取引先とのやりとり）
- タケシが読めなくて困ってる社内文書
- リサが添削してくれるシーン

### データ構造

```typescript
// Episode（ストーリー）
interface IzakayaEpisode {
  id: string;                    // "ep-001"
  title: string;                 // "ユキ、昇進がかかってる"
  subtitle: string;              // "Part 2: 間接回答の極意"
  targetPart: ToeicPart;         // "part2"
  targetSkill: string;           // "indirect-answers"
  targetScore: number;           // 目標スコアレベル (600, 700, 800...)
  story: StoryScene[];           // ストーリーシーン
  questions: ToeicQuestion[];    // TOEIC形式の問題
  masterTip: string;             // マスターの一言解説
  vocabHighlights: VocabItem[];  // 重要語彙
}

// ストーリーシーン
interface StoryScene {
  speaker: CharacterName;
  japanese: string;              // 日本語（ストーリー部分）
  english?: string;              // 英語（TOEIC問題に使う部分）
  mood: 'normal' | 'drunk' | 'angry' | 'excited' | 'thinking';
  action?: string;               // （ビールを飲みながら）
}

// TOEIC問題
interface ToeicQuestion {
  part: 1 | 2 | 3 | 4 | 5 | 6 | 7;
  audioScript?: string;          // リスニング原稿
  choices: string[];             // 選択肢 (3 or 4)
  correctIndex: number;
  explanation: string;           // 解説
  paraphraseMap?: {              // パラフレーズ対応表
    original: string;
    paraphrased: string;
  }[];
  trapType?: string;             // "verbatim" | "indirect" | "similar-sound" ...
  difficulty: 'easy' | 'medium' | 'hard';
  scoreLevel: number;            // この問題が解ければ何点レベルか
}

// キャラクター
interface IzakayaCharacter {
  id: string;
  name: string;
  nameEn: string;
  age: number;
  job: string;
  currentScore: number;
  targetScore: number;
  personality: string;
  catchphrase: string;           // 口癖
  weakPoints: ToeicPart[];       // 苦手パート
  strongPoints: ToeicPart[];     // 得意パート
  color: string;                 // テーマカラー
}
```

### UI/UXイメージ

**居酒屋の雰囲気をUIで表現**
- 背景: 温かみのある暗めのトーン（でもCLAUDE.mdのルールに従いクリーンに）
- ゴールド(#D4AF37)がメインカラー（提灯の光、ビールの色）
- 木目調のカード
- キャラアイコンは小さな丸アバター（既存パターンに合わせる）

**エピソード画面の構成**
1. ストーリーパート（日本語メイン、吹き出し形式）
2. 切り替わり「さて、ここからがTOEICの時間だ」（マスターのセリフ）
3. TOEIC問題パート（本番と同じ形式）
4. 解説パート（マスターの解説 + パラフレーズ対応表）
5. 振り返り（今日のスキルまとめ）

## 差別化ポイント（他のTOEIC教材にないもの）

1. **ストーリーの続きが気になる** -- 「ユキは800取れるのか？」が学習モチベーション
2. **キャラの失敗 = 自分の弱点** -- タケシが間違えるパターン = 学習者が間違えるパターン
3. **パラフレーズを物語で教える** -- 同じ出来事を違うキャラが違う言い方で話す
4. **酔いレベル = 難易度** -- シラフ→ほろ酔い→泥酔 = Easy→Medium→Hard
5. **居酒屋メニューがグラフィック問題** -- Part 3/4のグラフィック問題を自然に練習
6. **マスターの一言** -- 長い解説じゃなく、核心を突く短い一言
